Applicant Classification
Applicant First Name

The first name of the individual, must have a t least one if the Applicant is an individual.

  • Format is a String value up to 30 characters
Applicant Middle Name

The middle name of the individual, if this does not exist is it not required when the Applicant is an individual.

  • Format is a string value up to 30 characters
Applicant Last Name

The last name of the individual, must have at least one if the Applicant is an individual.

  • Format is a string value up to 30 characters
Applicant Suffix

The suffix of the individual, if this does not exist is it not required when the Applicant is an individual.

  • Format is a string value up to 30 characters
Applicant Name

The unparsed name of the legal entity.

  • Example; Smith Family Trust
Street Address (Mailing)

The mailing address with the address number, pre-directional, street name, post-directional, address unit designators and address unit value.
City (Mailing)

The name of the city of the Applicants mailing address.

  • Must be in a format of a String value with no more than 50 characters.
State (Mailing)

The two-character representation of the US state, US Territory, Canadian Province, Military APO FPO, or Territory of the Applicants mailing address.

Postal Code (Mailing)

The postal code (ZIP Code in the US) for the address. ZIP Code may be either 5 or 9 digits of the Applicants mailing address.

Country Code (Mailing)

The two-character representation of the US state, US Territory, Canadian Province, Military APO FPO, or Territory of the Applicants mailing address.

+ Add Additional Applicant

A link that, when selected, will add another section in the User Interface to add another Applicant to the transaction.
