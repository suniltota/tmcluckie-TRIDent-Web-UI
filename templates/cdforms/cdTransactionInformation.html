Borrower Classification

In any transaction type a Borrower is always present in the transaction, at least one. To structure the information correctly a borrower needs to be classified as either an individual or a Legal Entity (organization).

  • The user must choose a radio button;
  • The individual radio button is always the default.
Borrower Type

Is a role that Identifies the party in the transaction. A party may play multiple roles in a transaction. The enumerated default is the value of Borrower; however, some borrowers may or may not be on the Note so the borrower must have another designation.

  • For example, if only one party to the transaction is on the Note and both are on the Deed, then the user would choose the second borrower type as “Title Holder”.
  • Most transactions will just utilize the Borrower Type

Borrower Type

This is a role that Identifies the party in the transaction. A party may play multiple roles in a transaction. The enumerated default is the value of Borrower; however, some borrowers may or may not be on the Note so the borrower must have another designation.

  • In this instance of Borrower Type will always equal “Borrower”
Borrower First Name

The first name of the individual, must have a t least one if the borrower is an individual.

  • Considered Conditionally required if the borrower is an individual
  • Format is a String value up to 30 characters
Borrower Middle Name

The middle name of the individual, if this does not exist is it not required when the borrower is an individual.

  • Format is a string value up to 30 characters
Borrower Last Name

The last name of the individual, must have at least one if the borrower is an individual

  • Considered Conditionally required if the borrower is an individual
  • Format is a String value up to 30 characters
Borrower Suffix

The suffix of the individual, if this does not exist is it not required when the borrower is an individual.

  • Format is a string value up to 30 characters
Borrower Name

The unparsed name of the legal entity.

  • Example; Smith Family Trust
Street Address (Mailing)

The mailing address with the address number, pre-directional, street name, post-directional, address unit designators and address unit value.
City (Mailing)

The name of the city of the borrowers mailing address. Must be in a format of a String value with no more than 50 characters.
Country Code (Mailing)

The two-character representation of the US state,Canadian Province.
State (Mailing)

The two-character representation of the US state, US Territory, Canadian Province, Military APO FPO, or Territory of the borrowers mailing address
Postal Code (Mailing)

The postal code (ZIP Code in the US) for the address. ZIP Code may be either 5 or 9 digits of the borrowers mailing address.

Seller Classification

The Seller is only present in Purchase transactions. To structure the information correctly a Seller needs to be classified as either an individual or a Legal Entity (organization).

  • The user must choose a radio button;
  • The individual radio button is always the default.
Seller First Name

The first name of the individual, must have at least one if the Seller if the transaction is a purchase and the seller is an individual.

Lender Name

The unparsed name of the organization that is providing the credit.
Refinance Same Lender?

If the transaction is a refinance, this yes or no answer must be provided.

  • When true, indicates that the subject loan is refinanced with the current holder, current servicer or an affiliate of either.
  • Is required in all Refinance Transactions, when loan purpose is equal to refinance.
