Closing Information
General Information
Date Issued :
The date the integrated disclosure was mailed or delivered to the borrower as reflected on a single instance of the integrated disclosure document.
  • Must be in a format of DD/MM/YYYY or use the calendar icon to choose the date.
  • There is no constraint on this date as a user may need to create a .xml file to submit to an investor to generate an integrated closing disclosure document from the past
  • Is a Required field
Closing Date :
The date the closing documents were fully executed and the borrower becomes contractually responsible for the debt. Depending on the location (wet-sign, dry-sign, table-funded and/or escrow state) of the loan closing, the Closing Date may be the same date as either the Document Preparation Date and/or the Execution Date and/or the Recorded Date.
  • This date is defaulted to 5 days after the issue date if conducting data entry. Otherwise, the date will be rendered per the information in the upload process will be displayed
  • Must be in a format of DD/MM/YYYY or use the calendar icon to choose the date.
  • There is no constraint on this date as a user may need to create a .xml file to submit to an investor to generate an integrated closing disclosure document from the past.
  • Is a Required field
Disbursement Date :
The date the loan funds are disbursed to the borrower or another party on the borrower's behalf.
  • Must be in a format of DD/MM/YYYY or use the calendar icon to choose the date.
  • Constraint on the date as it cannot be a date prior to the issue date or the closing date.
  • Is a Required field
Settlement Agent Name :
The unparsed legal name of the Legal entity conducting the closing or settlement functions. This is typically an organization name.
  • Must be in a format of a string with no more than 100 characters.
  • Is a Required field
File # :
The identifying file number assigned to the transaction by the Closing Agent also known as a Settlement Agent.
  • Must be in a format of a String value with no more than 30 characters
  • Is a Required field

    Choose one: Street Address or Legal Description :
    In any transaction, the subject property location must be made clear and concise on the Closing Disclosure, thus the user interface provides two options:
    • Street Address
    • Legal Description, use if no street address exists.

    The address with the address number, pre-directional, street name, post-directional, address unit designators and address unit value

    The full, unparsed legal description of a parcel of real property.
    • Must be in a format of a String value with no more than 150 characters
    City Name
    The name of the city.
    • Must be in a format of a String value with no more than 150 characters

    The two-character representation of the US state, US Territory, Canadian Province, Military APO FPO, or Territory.
    Postal Code

    • This value is required regardless if of other property Information is available.

    Loan Information
    Loan Period, Type and Position
    Loan Term – No. of Years :
    In each transaction in accordance with the CFPB “Know Before You owe” Regulation must provide the entire term of the loan in years. Using the text box must input the number of years of the term.
    • Example, lenders typically use months so 360 months equates to 30 years.
    • Format is numeric up to 3 digits.
    Loan Term – No. of Months :
    In some transactions, such as construction to permanent or construction the term may be less than a year, so the user must input the number of months.
    • Example Construction Loan of 11 months.
    • Example Construction to Permanent with a single close would have both the number of years and the number of months potentially, ex: 30 years and 11 months.
    • Format is numeric up to 2 digits.
    Loan Type :
    Identifies the highest level private or public sector entity under whose guidelines the mortgage is originated.
    • Acceptable values are: Conventional | FHA | Local Agency | Other | Public and Indian Housing | State Agency | USDA Rural Development | VA.
    • Format are the only enumerated values above.
    Description :
    If the Other enumerated value is chosen for the Loan Type, provide a string value describing the loan type in the description field.
    Purpose:
    Specifies the purpose for which the loan proceeds will be used. This field is not editable as it is set when the user logs in to the application.
    • If the transaction needs to change the user must start a new transaction, cannot change the purpose.
    MIC# :
    The value for either Private Mortgage Insurance Certificate or the Agency Case ID if a government loan.
    • Format is string value with a character limit of 50.
    MERS MIN Identifier :
    The value of the MERS registered Loan, if applicable.
    • If this exists then the user should provide the value of the MERS MIN registered id for the transaction to include in the GSE XML file.
    Lien Position :
    Specifies the priority of the lien against the subject property.
    • Format is enumerated values that include the following: First Lien | Fourth Lien | Second Lien | Third Lien
    Lender Loan Identifier :
    The unique loan identifier for a specific transaction for a specific lender.
    • Format is a string value up to 30 characters.
    • This is a required value and must be present in the transaction.
    Appraisal ID :
    A unique identifier assigned for all appraisal data delivered for this transaction.
    • Format is a String Value up to 150 characters.
    • This field is conditionally required, if it exists input the value.
    Manual Underwriting Flag :
    In every transaction, must Indicate if the loan was manually underwritten.
    • Is required for every transaction.
    • Format is a Boolean value using a yes/no drop-down box.
    AUS Type :
    AUS stands for Automated Underwriting System, the user must provide the value when the Manual Underwriting Flag is NO.
    • Fannie Mae requires a DU AUS Type and the associated ID, if it is not present in the transaction when submitting the UCD XML file Fannie Mae will provide a UCD Identifier.
    • Freddie Mac will accept any AUS Value and ID.
    • Format is a set of Enumerated Values, if the value is not present the use can chose other. If Other is chosen then a description field will be provided for that instance and the user may type in the name of the AUS used.
    • Can support up to 9 different AUS systems (instances).
    AUS Identifier :
    A unique identifier assigned for all appraisal data delivered for this transaction.
    • Format is a String Value up to 150 characters.
    • This field is conditionally required, if it exists input the value.
    Add Additional Automated Underwriting System Names and Identifiers link :
    Only 3 AUS Type and AUS Identifiers show in the user interface, if more are needed a user must click the link to add 3 more instances, can have up to 9 total AUS System Names and Identifiers.
    Product Description And Information
    Product Description :
    A description of the loan product which includes, when applicable, certain loan features added to the loan product that may change the periodic loan payment.
    • Required for every transaction.
    • Value is driven by the CFPB’s “Know Before You Owe” regulation.
    • Format is a String Value up to 50 Characters.
    Amortization Type:
    A classification or description of a loan or a group of loans generally based on the changeability of the rate or payment over time.
    • Required for every transaction.
    • Format is a list of enumerated values that include: Adjustable Rate | Fixed | GEM | GPM | Graduated Payment ARM | Step
    Adjustment Information, required when the Amortization Type does not equal Fixed :
    Provide the number of months for the fixed period :
    The number of months after origination in which the first interest rate adjustment occurs, if the interest rate on the subject loan can change.
    • Format is numeric up to 3 characters.
    Provide the life of loan Cap Rate:
    The stated maximum percentage “ceiling rate” to which the interest rate can increase over the life of the loan.
    • Format is a percent 3.4.
    Provide the number of months first change period :
    The number of months between rate adjustments for the first changed after the fixed period, if the interest rate on the subject loan can change.
    • Format is numeric 3.
    Provide the first change interest rate limit :
    The maximum number of percentage points by which the rate can increase from the previous interest rate for the First Change after the fixed period.
    • Format is a percent 3.4.
    Provide the number of months subsequent change period :
    The number of months between rate adjustments for the subsequent changes after the first period change from the fixed period, if the interest rate on the subject loan can change.
    • Format is numeric 3.
    • At times the number of months between the interest rate changes can be different from the first change after the fixed period.
    Provide the subsequent change interest rate limit :
    The maximum number of percentage points by which the rate can increase from the previous interest rate for the Subsequent changes after the first change after the fixed period.
    • Format is a percent 3.4.
    If Amortization Type is Step, Provide the Total Step Count :
    For a loan with a step provision, the total number of step rate adjustments allowed under the step agreement.
    • Format is a numeric 3.

    Loan Terms
    Loan Terms Information
    Loan Amount :
    Enter the value of the loan or upload the amount through a template or file.
    • Format is an amount 9.2
    • The amount will be either the Note amount which is the amount to be repaid as disclosed on the note - If the Interest rate and note amount are known at consummation
    • OR
    • The Fully indexed principal and interest amount which is the dollar amount of the principal and interest payment when calculated using the loan amount and fully-indexed interest rate (in place of Note Interest Rate) to arrive at full amortization during the loan term. If the Interest Rate and Note amount are NOT known at consummation.
    • The system defaults the yes/no drop-down box to NO for the question “Does this change after closing?” using the information from the product description section (negative amortization indicator = false is no in the drop down).
    • The User may change the yes/no drop down
    • User May enter in the Dollar amount the bullet stating “Can go as high as” for the loan amount maximum limit, otherwise it will pull from the product description section under negative amortization.
    • User May enter the number of months in the text box next to the Statement “Can Increase Until Year” for the second bullet, or the system will pull in the values from the product description section under Negative amortization.
    Interest Rate :
    Enter or Upload the interest rate as it will be disclosed on the note or the Fully Disclosed Index Rate.
    • If manually entered interest rate, the system defaults for the question “Does this change after closing?” Yes | No drop-down box to NO until the user completes the Product Description section.
      • If the Amortization Type in the product description section is equal to Fixed, then the “Does this change after closing?” remains No
      • If the Amortization Type in the product description section is NOT Equal to Fixed, then the “Does this change after closing?” changes to Yes with additional bullet points to be filled in using the information from the Product Description section, however these fields are editable by the user.
      • The following fields must have values if the Amortization Type is NOT fixed:
        • Adjusts every ______ years starting in year______ - Provide the number of Years for the adjustment AND the year in which the adjustment will start plus one month because interest changes in arrears. So if the interest rate is due to change in month 60 (aka 5 years) then the form needs to state in year 6 because the payment will actually change in year 6.
        • Can go as high as _____ in year ________ - The can go as high as statement is the ceiling rate for the life of the loan, then in year again is the year in which the ceiling will be reached.
    • Regardless of Amortization Type the user must provide the values for; current rate set date, the loan price quote and the Average Prime Offer Rate (APOR) in the 3 text boxes allotted for the values.
      • Current Rate Set Date is the Date on which the interest rate for the loan was set by the lender for the final time before closing.
      • Loan Price Quote is the mortgage loan interest rate for which the price quote is calculated.
      • The APOR is an annual percentage rate that is derived from average interest rates, points, and other loan pricing terms currently offered to consumers by a representative sample of creditors for mortgage transactions that have low-risk pricing characteristics. The Federal Reserve Board publishes average prime offer rates for a broad range of types of transactions in a table updated at least weekly. This rate is used in determining Higher Priced Mortgage Loans (HPML).

    Projected Payments
    Years :
    Each column must be expressed in whole years
    • Years 1 - 30
    • Years 1 - 5, etc.….

    Whole years are defined as twelve-month intervals that begin on the due date of the initial periodic payment. When the same Estimated Total Periodic Payment amount is valid for several years before a payment change event occurs, then that Year Range is identified as a column heading.

    The entire loan term must be represented by the Year Range Columns that can only have a maximum of 4 columns in the transaction.

    See the Know Before You Owe regulation for the rules of what drives a payment adjustment.
    Add Additional Column Link :
    The Projected Payment section is driven off several pieces of information:
    • Product Description – Amortization Type if Fixed then typically only one column or multiple columns of the Amortization Type is Adjustable Rate (ARM) or Not Fixed.
    • Upload of multiple instances of data will automatically provide the additional columns with values.

    If the user is manual data entry and requires additional columns, then the user may click on the Add Additional Column Link that will add 3 additional columns one at time up to a total of total columns.
    Principal and Interest Payment :
    In each column (1-4) a Minimum amount and Maximum amount text boxes are available to the user for input.
    • Minimum Amount - The minimum amount of the principal and interest payment during the projected payment period. Typically this is not used for the first column due to any fixed periods would be identified in the first period.
    • Maximum Amount - The maximum amount of the principal and interest payment during the projected payment period.
    Mortgage Insurance – Is required? :
    This information is required for every transaction, must identify if there is MI present in the transaction.
    • If NO, then nothing else needs to be done
    • If Yes, provide the following information that is found under the Mortgage Insurance Label in the far-left column
      • No. Months Duration?
      • Scheduled Termination Date
      • MI Company Name
      • MI Company Name Description, if other
    • If yes, provide the amount of MI this will drive the need for additional columns due to displaying the amount per change period until the MI is no longer required.

    Estimated Total Monthly Payment Amount :
    Every Column will have a total for both the minimum amount and the maximum amount, this is not an editable field and the system provides the value to the user in the user interface.
    Estimated Taxes, Insurance and Assessment of Projects Payments
    $ amount per month :
    This amount is calculated via the Calculate button.
    Check Boxes – Property Taxes and Homeowners Insurance:
    The user may select neither, both or just one.
    Check box – Other :
    The user may enter one or many Other items, the purpose of adding many when the Closing Disclosure only has one on the PDF if for support of the GSE requirements in the UCD XML file.
    • Format is a list of Enumerated values to choose.
    • If other is Chosen, then the Description box will be enabled for the user to input the value.
    In Escrow? :
    The user may select from a drop down whether the item is in an escrow account or not.
    • Format is a drop-down box with Yes or No.
    • Will default based on the information in Closing Cost Details Section G. Initial Escrow at Closing.
    Add Additional Item :
    This link will all you to add another row to choose from the drop down and indicate if in escrow or not in escrow.

    • Help search must sear on “Costs at Closing”.

    Closing Cost Details (Fee Page)
    A. Origination Charges
    Section Header Name :
    A unique identifier assigned for all appraisal data delivered for this transaction.
    • Format is a String Value up to 150 characters.
    • This field is conditionally required, if it exists input the value.
    Section Total Amount Borrower Paid :
    This provides the user an ability to view the total amount of the section, this is automatically updated once you have entered in the values for a record in either the Paid By Borrower At Closing or Before Closing
    *Percentage * Loan Amount (Points) – Label :
    In every transaction, the line 01 item called Loan Amount (Points) is provided on the form. The transaction may or may not have discount points.
    The Label name “Loan Amount (Points)” is referred to as predefined text by the CFPB.
    • If NO, “0” must be disclosed in the Label section.
    • If YES, both the percentage of the Loan Amount must be disclosed in the label section.
    All Rows must include the following:
    Fee Label Name :
    A free-form text field that provides a human-readable, display-ready representation for the corresponding enumerated value.
    • Typically, the name a lender calls a specific fee, the user would type that fee in this location.
    • Format is a String Value up to 150 characters.
    Fee Type :
    Specify the general type of fees associated with a real estate finance transaction.
    • Required to support the GSE xml file, even though a Fee Label Name is provided, the user must choose an enumerated value to associate to the label name.
    • If other is chosen, then the user will be provided another text box (see below).
    Fee Type Other Description :
    This field is required to be filled-in if the Fee Type is Other.
    • Text box for a String Value that allows a user to type in any characters to meet the need of the Fee Type.
    • Recommended to use as many enumerated values as possible when sending the record to the GSEs.
    Full Name :
    If applicable see the CFPB “Know Before You Owe” regulation when to include this value in the Closing Disclosure.
    • When required, include the unparsed name of either the organization or legal entity.
    Fee Paid To Type :
    Identify the category or type of payee to which the fee or payment will be paid.
    • Format is a list of enumerated values that include: Broker | Investor | Lender | Other | Third Party Provider.
    • Must be provided when there is a fee entered in the row.
    Borrower Paid At Closing :
    The amount of the fee that is paid by the Borrower at closing.
    • Format is amount 9.2.
    • Cannot be negative.
    • Is present only if there is a Fee Type chosen for the row.
    Borrower Paid Before Closing :
    The amount of the fee that is paid by the Borrower before closing.
    • Format is amount 9.2.
    • Cannot be negative.
    • Is present only if there is a Fee Type chosen for the row.
    Seller Paid At Closing :
    The amount of the fee that is paid by the Seller at closing.
    • Only included in a Purchase Transaction, is not present when using the Alternate Form for a Refinance transaction.
    • Format is amount 9.2.
    • Cannot be negative.
    • Is present only if there is a Fee Type chosen for the row.
    Seller Paid Before Closing :
    The amount of the fee that is paid by the Seller before closing.
    • Only included in a Purchase Transaction, is not present when using the Alternate Form for a Refinance transaction
    • Format is amount 9.2.
    • Cannot be negative.
    • Is present only if there is a Fee Type chosen for the row.
    Paid By Other :
    The amount of the fee that is paid by a Third Party or the Lender.
    • Format is amount 9.2.
    • Cannot be negative.
    • There is NOT a “At Closing” or “Before Closing” indication.
    • Is present only if there is a Fee Type chosen for the row.
    Paid By Lender Flag :
    When a value is in the Paid By Other column, if the amount is paid by the Lender then the user must check the Paid By Lender Flag
    • Required to indicate if a Lender is paying a Fee per the rule.
    Include in APR Calculation :
    For each record included must Indicate if the fee is included in the APR calculation.
    • The annual percentage rate for the loan based on the accepted industry standard defined by Regulation Z (Section 12 C.F.R. 1026.18(e)) used for Qualified Mortgages and general disclosure purposes; if questions regarding this regulation must see the CFPB website.
    • Format is a Boolean value, using a drop-down box with a Yes or No value.
    Include in Flag Z Points/Fee :
    When true, indicates that item is included in the Regulation Z Points and Fees calculation.
    • Must be included for all Fees except for Tax related fees.
    • Format is a Boolean value, using a drop-down box with a Yes or No value.
    Additional Lines can be added. :
    To add more fees to the section the user may click on the link to add another row.
    Trash Symbol :
    The Trash Can symbol at the end of the record will delete the entire row including all data.
    Clear Symbol :
    The Clear symbol at the end of the record will remove all the data but leave the row in tact if a user wants to remove the information in the cells all at once time.
